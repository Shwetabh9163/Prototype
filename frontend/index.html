How Each Algorithm Works
⟳ Round Robin
Requests rotate sequentially: S1→S2→S3→S1→... Creates perfectly even distribution. Used by Nginx by default. Best when servers have equal capacity.
📊 Least Connections
Routes to the server with fewest active connections. Adapts dynamically to server speed differences. Better for requests with variable processing time.
🔑 IP Hash
User's IP deterministically maps to one server. The same user always hits the same server (session persistence). Used when stateful apps can't share sessions.

Token Bucket Algorithm Explained
How It Works
1. A bucket starts with N tokens (your limit)
2. Each request consumes 1 token
3. Tokens refill at a fixed rate (e.g. 2/sec)
4. If bucket is empty → HTTP 429 Too Many Requests
5. Allows short bursts up to bucket capacity
Real-World Usage
Twitter API: 500,000 tweets/month Basic tier
GitHub API: 5,000 requests/hour authenticated
Cloudflare: Rate limit rules per IP/path
Redis implementation: INCR + EXPIRE (atomic)
Response: HTTP 429 + Retry-After header
